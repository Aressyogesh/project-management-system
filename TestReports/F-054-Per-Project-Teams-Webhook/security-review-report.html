Security Review Report — F-054 Per-Project Teams Webhook Configuration

Reviewer: Claude Sonnet 4.6  |  Date: 2026-07-15  |  OWASP Top 10 check

SEC#OWASP CategoryCheckStatusNotes
SEC-1 A01 Broken Access Control Only PM / Admin / Super can save/clear webhook URL PASS Service checks SystemRole first; then projectMember role. ForbiddenException thrown for DEVELOPER, QA, etc. No @Roles on controller — auth enforced in service.
SEC-2 A02 Cryptographic Failures Webhook URL stored as plaintext — acceptable PASS Teams webhook URLs are not secrets (they are HMAC-signed by Microsoft). Storage as VARCHAR(2000) is standard practice.
SEC-3 A03 Injection No SQL injection — Prisma parameterised queries only PASS teamsWebhookUrl is passed as a Prisma parameter, never interpolated into raw SQL.
SEC-4 A03 Injection (SSRF) Backend does NOT call teamsWebhookUrl — only stores it PASS Backend sends the URL to ActivePieces, not to Teams directly. AP makes the outbound call. SSRF risk is in AP, not PMS backend.
SEC-5 A04 Insecure Design URL validated as HTTPS-only PASS @IsUrl({ require_protocol: true, protocols: ['https'] }) rejects HTTP URLs. Prevents accidental plaintext webhook endpoints.
SEC-6 A05 Security Misconfiguration teamsWebhookUrl not logged PASS AutomationService only logs error messages, not the URL value. ProjectsService does not log the webhook URL.
SEC-7 A07 Auth Failures JwtAuthGuard covers all endpoints globally PASS New PATCH :id/teams-webhook is automatically protected by the global JwtAuthGuard. No @Public decorator added.
SEC-8 A08 Software Integrity No new external dependencies added PASS Uses existing class-validator (IsUrl), Prisma, NestJS. No new npm packages.
SEC-9 A09 Logging Errors logged but URL value not exposed PASS All automation .catch() blocks log err.message only, not the full URL or body.
SEC-10 A10 SSRF (frontend) Frontend input field does not make direct outbound requests PASS Frontend PATCH call goes to PMS backend only. The webhook URL is never called from the browser.
SEC-11 XSS Webhook URL rendered in input value only (not innerHTML) PASS teamsWebhookUrl displayed via React controlled input value prop. No dangerouslySetInnerHTML usage for this field.
11 / 11 PASS — No security issues found
