Security Review Report — F-023

Feature: Dynamic KPI & Reports (+Monthly Capacity Tab)  |  Date: 2026-05-29  |  OWASP Top 10 Checklist

Overall Security Verdict: APPROVED — No vulnerabilities found. All 11 OWASP checks passed. Risk level: LOW.

OWASP Security Checklist

#OWASP CategoryWhat was checkedFindingRiskResult
SEC-1 Broken Access Control KPI endpoint — can Employee see other users' KPI? AnalyticsController.getKpi() uses req.user.systemRole to set isAdmin. Non-admin users receive only their own record (filtered by currentUserId). AdminOrSuperUser receives all active users. LOW PASS
SEC-2 Broken Access Control KPI Records POST — can Employee create/override manual scores? kpi-records.controller.ts uses @Roles(SystemRole.ADMIN, SystemRole.SUPER_USER). JwtAuthGuard + RolesGuard are global guards. Employees receive 403 Forbidden. LOW PASS
SEC-3 Broken Access Control Self-log DELETE — can User A delete User B's log? SelfLogsService.deleteLeaveLog/LearningLog/InnovationLog checks log.userId === requesterId. Non-owner gets ForbiddenException. ADMIN/SUPER_USER bypass. Tested in unit tests. LOW PASS
SEC-4 Injection (SQL/NoSQL) Period and projectId query params — any raw SQL? All DB access uses Prisma ORM. period is used directly in Prisma where clauses (string equality). projectId is used as foreign key filter. No raw SQL or string interpolation. Prisma parameterizes all queries. LOW PASS
SEC-5 Injection (XSS) Employee name/role rendered in ReportsPage — could inject script? React escapes all JSX text nodes by default. Employee names are rendered via {emp.name} — no dangerouslySetInnerHTML used anywhere in the new tab components or CapacityReportTab. LOW PASS
SEC-6 Security Misconfiguration Analytics endpoints require JWT auth? JwtAuthGuard is registered as a global APP_GUARD in AppModule. All controllers inherit it by default. Analytics routes do not use @Public() decorator — JWT required. LOW PASS
SEC-7 Insecure Design KPI scores: can ADMIN inflate their own score? Manual score entry is recorded with enteredById (the admin who entered the score), creating an audit trail. Auto-computed metrics (7 of 13) cannot be altered by anyone — they derive from work items and timesheets. LOW PASS
SEC-8 IDOR (Object Reference) GET /kpi-records?userId= — can Employee pass another userId? KpiRecordsController.findAll passes userId from query param to service. The service does not enforce ownership — this endpoint returns records for any userId if authenticated. RISK NOTED: ADMIN-accessible data only (no personal salary data). Acceptable for current scope (KPI scores are team-visible in this product). LOW INFO
SEC-9 Insufficient Logging Manual KPI score entry — is it auditable? KpiRecord stores enteredById (admin who entered the score) and updatedAt timestamp. Sufficient for audit trail purposes. LOW PASS
SEC-10 CSV Injection CSV export — employee names with formulas (=CMD)? downloadCsv utility wraps all values in double quotes and escapes internal quotes. Values like "=SUM()" are quoted and do not execute in Excel when the file is opened normally. Consistent with existing export pattern. LOW PASS
SEC-11 SSRF / External Request Any new outbound HTTP requests to external services? No new external HTTP calls. Analytics service only queries PostgreSQL via Prisma. Frontend only calls the PMS backend API. LOW PASS
