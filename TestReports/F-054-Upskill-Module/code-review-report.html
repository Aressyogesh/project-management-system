CR-01 — Error Instead of BadRequestException for Missing File BLOCKING FIXED

backend/src/upskill/upskill.controller.ts:135

A — Line-by-line diff scan

The Multer fileFilter error callback used new Error(...) instead of NestJS BadRequestException. A bare Error thrown inside Multer's callback is not caught by NestJS's exception filters and results in an unhandled exception / 500 response instead of a 400 Bad Request with the standard error envelope.

Imported BadRequestException and used it in the fileFilter callback and in the missing-file guard.

CR-02 — Upload Directory Not Created on Deploy WARNING FIXED

backend/src/upskill/upskill.controller.ts:35

B — Removed-behavior auditor / Deployment concern

The uploads/upskill/ target directory did not exist on a fresh Render deploy. Multer would throw ENOENT on the first file upload attempt, causing a 500 error. The existing main.ts only mkdirSyncs avatars and images.

Added mkdirSync(UPSKILL_UPLOAD_DIR, { recursive: true }) at module-level in the controller so the directory is created at startup.

CR-03 — rejectAssignment Incorrectly Set approvedById WARNING FIXED

backend/src/upskill/upskill.service.ts:219

A — Line-by-line diff scan

The initial implementation of rejectAssignment included approvedById: rejectorId in the Prisma update payload. This incorrectly marks the rejector as an "approver", corrupting the audit trail and causing false-positive reads in findApprovedForUserInPeriod (used by the analytics/KPI scoring service).

Removed approvedById from the reject update payload. Only status: REJECTED and rejectionReason are set.

CR-04 — UpskillAssignmentSection Queried Without period Filter WARNING FIXED

frontend/src/features/kpi/pages/KpiPage.tsx

C — Cross-file tracer

UpskillAssignmentSection used ['upskill-mine', period] as the React Query cache key (period-aware) but called upskillApi.listAssignments({ mine: true }) without passing period. This caused all assignments across all months to render regardless of the KPI period selected, and cache invalidations for different periods would share the same stale data.

Updated the API call to listAssignments({ mine: true, period }); updated upskillApi.ts, the controller query param, and buildPeriodFilter in the service to support the period param.
