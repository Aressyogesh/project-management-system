SEC-1 Injection (SQL, Command) All DB queries use Prisma parameterized queries. No raw SQL strings. No shell command execution. search filter uses Prisma contains with mode: insensitive — no string concatenation into queries. PASS
SEC-2 Broken Authentication All board endpoints protected by JwtAuthGuard globally. No anonymous access to work item data. Sprint management requires authenticated session. Token validation handled by existing auth module — no changes made. PASS
SEC-3 Sensitive Data Exposure Work item selects only necessary user fields (id, fullName, profilePhoto). Reporter/assignee data never includes email or password hash in board responses. TimesheetEntry user select limited to id/fullName. PASS
SEC-4 Broken Access Control RBAC enforced: only assignee/reporter/PM/admin can edit items. Only PM/Admin can delete items. Only PM/Admin can manage sprints. TimesheetEntry creation restricted to assignee/reporter. ForbiddenException thrown for unauthorized actions. PASS
SEC-5 Security Misconfiguration No debug endpoints exposed. No hardcoded credentials in new code. Position field uses integer (no float precision issues). Cascade deletes correctly configured to prevent orphan data. PASS
SEC-6 XSS (Cross-Site Scripting) All user content rendered via React JSX (auto-escaped). No dangerouslySetInnerHTML usage. Work item title/description rendered as text nodes. Comment content not rendered as HTML. PASS
SEC-7 Insecure Deserialization NestJS DTOs use class-validator for input validation. No custom deserialization logic. Prisma handles type coercion. WorkItem labels/components validated as string arrays. PASS
SEC-8 CSRF API uses JWT Bearer tokens in Authorization header (not cookies), making CSRF attacks non-applicable. State-mutating endpoints use appropriate HTTP verbs (POST, PATCH, DELETE). PASS
SEC-9 Privilege Escalation Project role checked against project membership table at request time — not cached in JWT. Server-side validation prevents client-side role spoofing. RBAC matrix enforced consistently across all endpoints. PASS
SEC-10 Insecure Direct Object Reference (IDOR) Work item projectId verified via ProjectRoleGuard before accessing. TimesheetEntry resolves workItem.projectId for membership check. Sprint resolved to projectId before write operations. PASS
SEC-11 Component Vulnerabilities @hello-pangea/dnd is maintained fork of react-beautiful-dnd with no known CVEs. No new direct dependencies with known vulnerabilities. All other dependencies unchanged from previous features. PASS
