Security Review Report — F-029: Kanban Board Enhancements

Reviewer: Claude Sonnet 4.6  |  Date: 2026-06-01  |  OWASP Top 10 + 1 additional check

Security Checks (SEC-1 through SEC-11)
IDCheck (OWASP)FindingStatus
SEC-1 A01: Broken Access Control All new endpoints (board-column-configs, test-cases) are protected by the global JwtAuthGuard. No anonymous access possible. BoardColumnConfig and TestCase operations are scoped to projectId/workItemId — users cannot access items from other projects via these endpoints. PASS
SEC-2 A02: Cryptographic Failures No new cryptographic operations introduced. All existing JWT/bcrypt mechanisms remain unchanged. Test case data and column config data are not sensitive (no PII, no credentials). PASS
SEC-3 A03: Injection All DB operations use Prisma ORM with parameterised queries. No raw SQL or string interpolation in DB queries. The milestoneId, workItemId, and projectId route/query params are passed directly to Prisma where clauses — no concatenation into SQL strings. PASS
SEC-4 A04: Insecure Design Test case creation requires an authenticated userId (from JWT sub claim). Board column label length capped at VarChar(100) in DB and validated via @MaxLength in DTO. Test case fields have appropriate VarChar limits. PASS
SEC-5 A05: Security Misconfiguration New modules (TestCasesModule, BoardColumnConfigsModule) are registered in AppModule. No new environment variables, no new CORS exceptions, no debug endpoints introduced. PASS
SEC-6 A06: Vulnerable Components No new npm packages added. Existing react-query, @hello-pangea/dnd, and prisma libraries unchanged. PASS
SEC-7 A07: Authentication Failures JWT guard is global — all new routes inherit authentication. No public decorators added to new controllers. PASS
SEC-8 A08: Software Integrity No new build pipeline changes. No new CDN scripts or external resources loaded. GlobalProgressBar uses inline CSS keyframe animation — no external stylesheet. PASS
SEC-9 A09: Logging Failures NestJS default logger covers incoming requests. No new sensitive data (test steps, bug details) are logged to console. PASS
SEC-10 A10: SSRF No server-side URL fetching introduced. All new API calls are internal Prisma queries only. PASS
SEC-11 XSS (Frontend) Test case content (title, steps, actualResult) is rendered via React's JSX (text nodes), not dangerouslySetInnerHTML. Column labels rendered as text. Client names in link hrefs use React router's <Link to={...}> — no raw anchor href interpolation. Board description fields may render rich text via existing mechanism — unchanged in this feature. PASS
