Additional Security Observations

  • Input Length Validation: All new string fields in DTO have @MaxLength() decorators (module: 100, build versions: 50). Enum fields use @IsEnum(). UUID fields use @IsUUID(). Validated at transport layer by NestJS global ValidationPipe.
  • Mass Assignment: Not a risk — Prisma create()/update() calls explicitly list each field; no spread of request body directly to Prisma data object.
  • Cross-Project Data Leakage: Milestone selectors are populated from milestones fetched via milestonesApi.list(projectId) — project-scoped. A user cannot select a milestone from a different project through the UI. Server-side, the releaseMilestoneId FK enforces referential integrity at DB level.
  • Frontend State Isolation: Each WorkItemModal instance fetches its own detail via queryKey: ['workItem', item.id] — no cross-item state bleed.
