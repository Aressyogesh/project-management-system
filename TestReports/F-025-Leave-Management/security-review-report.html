Additional Security Observations

  • Input Length Validation: reason and approvalNote fields have @db.VarChar(500) at DB level and @MaxLength(500) in DTO — prevents excessively large payloads from reaching the database.
  • Date Injection Prevention: startDate and endDate are stored as @db.Date (Prisma validates ISO date string before writing). Server-side endDate < startDate check prevents logical inversions.
  • Mass Assignment: Not a risk — prisma.leaveRequest.create() explicitly enumerates each field; request body is not spread directly into the Prisma data object. DTOs use class-validator to whitelist permitted fields.
  • Cross-User Data Leakage: Employee list endpoint returns only the calling user's records. Admin cannot be impersonated: systemRole is read from the JWT payload signed by the server, not from a user-supplied header or query param.
  • Capacity Report Exposure: The /analytics/reports/capacity endpoint should be restricted to ADMIN/SUPER_USER (existing guard in analytics controller). Leave data for all employees is included — verified that non-admin callers cannot reach this endpoint.
